# Лекц №6 "Төвөгтэй байдлыг бууруулах, Дизайны дотоод шинж чанар"

## Сэдэв: 
Cohesion (Дотоод нягт холбоо)

---

## Тодорхойлолт

Cohesion гэдэг нь праграмм хангамжийн нэг модуль (класс эсвэл функц) доторх элементүүд нь нэг л зорилгын төлөө хэр зэрэг нягт ажиллаж байгааг илэрхийлдэг үзүүлэлт юм. Өөрөөр хэлбэл, тухайн модуль доторх бүх код нь "нэг л сэдэвт" хамааралтай байхыг хэлнэ.

## Гол санаа

-Cohesion өндөр байх тусам системийн дизайн сайн байна.
-Модуль бүр нэг л тодорхой үүрэг хариуцах ёстой.
-Өндөр нягт холбоотой код нь ойлгоход амар, засварлахад хялбар, алдаа гарах магадлал багатай байдаг.

## Жишээ

Сайн жишээ (Өндөр Cohesion):
ReportGenerator класс нь зөвхөн тайлантай холбоотой үйлдлүүдийг хийнэ.
class ReportGenerator {
  fetchData() {}      // Тайлангийн өгөгдөл цуглуулах
  formatTable() {}    // Хүснэгт хэлбэрт оруулах
  exportToPDF() {}    // PDF болгож гаргах
}

Муу жишээ (Бага Cohesion):
GeneralManager класс дотор хоорондоо хамааралгүй зүйлс холилдсон.

JavaScript
class GeneralManager {
  calculateTax() {}   // Татвар бодох
  saveUserFile() {}   // Файл хадгалах
  sendSmsAlert() {}   // SMS мэдэгдэл илгээх
}

## Зүйрлэл

Багаж хэрэгслийн хайрцаг:

-Өндөр Cohesion: "Мужааны хайрцаг" дотор зөвхөн модтой ажиллахад хэрэгтэй алх, хөрөө, хадаас, хэмжүүр байхтай адил юм. Бүгд нэг зорилготой.

-Бага Cohesion: Хэрэв багажны хайрцаг дотор алхнаас гадна гал тогооны халбага, шүдний сойз, машины дугуй байвал энэ нь ямар ч ашиглахад төвөгтэй, замбараагүй байх болно.
